# Release checklist

Before publishing a new release:

1. Update `CUSTOM_VERSION` in `LegalNoticeFooterModule.php`.
2. Update `CUSTOM_RELEASE_DATE` in `LegalNoticeFooterModule.php`.
3. Update `latest-version.txt`.
4. Update release-related content in `README.md`, if needed.
5. Check all `.po` and `.mo` files for obvious translation or formatting errors.
6. Run the relevant syntax and translation checks.
7. Commit the release changes.
8. Create and publish the GitHub release.
